## Key Design Decisions Made

1. **Unified State System** — 6 states everywhere: Default, Hover, Press, Focus, Disabled, Danger
2. **Error renamed to Danger** — consistent with Button naming
3. **Press = :active in CSS** — Figma "Press" maps to CSS `:active`
4. **Disabled uses explicit tokens** — `action/disabled` + `action/disabled-foreground`, NOT opacity:0.5
5. **Check indicators on LEFT** — all menu items (DropdownMenuItem, SelectItem, ComboboxItem) use left-side check, matching DuBois convention
6. **Inner component pattern** — .MenuLabel handles content (label + description + icon), .MenuTrailing handles right slot (none/icon/text), .MenuRow wraps everything for compositions
7. **20px height containers** — icons and indicators wrapped in 20px frames to vertically center with text line height
8. **Size variants on building blocks** — .ActionLabel and .InputContent have Size=Default/Small with different gaps (8px/4px)
9. **No CommandMenu** — shadcn Command component has no DuBois equivalent, removed from Figma
10. **Compositions use .MenuRow** — DropdownMenu and SelectDropdown have 10 .MenuRow slots, designers switch Type variant per row
